# ReMind Demo 稿：記憶功能展示

---

## ⚡ 20 秒版（超濃縮）

> 「ReMind 是一個**會記住你的 AI**。
> 
> 你告訴它『我叫小明，喜歡籃球』，過一陣子問它，它能正確回答。
> 
> 重點是——**每個人的記憶完全隔離**，別人問不到你的資訊。
> 
> 這就是 RAG + 向量資料庫的力量。謝謝。」

---

> **展示時間**：約 5 分鐘  
> **展示重點**：記憶儲存 → 記憶回憶 → 多用戶隔離

---

## 🎬 開場白（30 秒）

> 「大家好，我們是 ReMind 團隊。今天要展示的是一個**具備長期記憶的 AI 陪伴者**。
> 
> 一般的 AI 只能記住當前對話，但 ReMind 不一樣——它能**跨越時間記住你說過的每一件事**，就像一個真正了解你的朋友。」

---

## 📌 Demo 流程

### 第一幕：建立記憶（1 分鐘）

**場景**：用戶 A 與 ReMind 第一次對話

```
用戶 A：「我叫小明，最喜歡打籃球，明天要去面試」
```

**預期回應**：Bot 溫暖回應，確認收到這些資訊

**旁白**：
> 「小明告訴 ReMind 三件事：名字、興趣、明天的行程。
> 這些資訊會被自動提取為『事實』，存入我們的向量資料庫。」

---

### 第二幕：記憶回憶（1.5 分鐘）

**場景**：等待 15 秒後，測試記憶

```
用戶 A：「我叫什麼名字？」
```
**預期回應**：「你是小明呀！」

```
用戶 A：「我喜歡什麼運動？」
```
**預期回應**：「你喜歡打籃球！」

```
用戶 A：「我明天要做什麼？」
```
**預期回應**：「你明天要去面試，加油！」

**旁白**：
> 「可以看到，ReMind 能正確回憶剛才說過的每件事。
> 這是透過 **RAG（檢索增強生成）** 技術實現的——
> 當用戶提問時，系統會先搜尋向量資料庫，找到相關記憶，再生成回應。」

---

### 第三幕：多用戶隔離（1.5 分鐘）⭐ 核心功能

**場景**：換成用戶 B 登入

```
用戶 B：「小明喜歡什麼運動？」
```
**預期回應**：「抱歉，我不知道小明是誰耶。」

```
用戶 B：「告訴我關於小明的事」
```
**預期回應**：「我沒有關於這個人的任何記憶。」

**旁白**：
> 「這是我們系統最重要的安全功能——**記憶隔離**。
> 
> 每一筆記憶都綁定了 `user_id`，用戶 B 完全無法讀取用戶 A 的任何資訊。
> 這確保了每個用戶的隱私都受到保護。」

---

### 第四幕：情感化 UI（30 秒）

**展示重點**：
- Embed 卡片的**紫色側邊條**
- 底部的 **Footer 文字**
- （可選）不同情緒的動態色彩變化

**旁白**：
> 「我們也很注重使用者體驗。
> 所有回覆都使用 Discord 的 Embed 卡片格式，
> 配上我們設計的主題色和標誌性的 Footer，
> 讓對話更有質感。」

---

## 🎯 結語（30 秒）

> 「總結一下 ReMind 的三大特色：
> 
> 1. **長期記憶**：利用向量資料庫 + RAG 技術，記住每一個重要細節
> 2. **隱私安全**：透過 Metadata Filter 實現多用戶記憶隔離
> 3. **情感化設計**：Embed UI + 動態色彩，打造有溫度的陪伴體驗
> 
> 謝謝大家！歡迎提問。」
