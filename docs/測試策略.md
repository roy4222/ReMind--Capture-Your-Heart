📋 5 大測試項目
✅ 測試 1: 模型穩定性 + UI 呈現（5 分鐘）
目的: 確認模型切換成功 & Embed 卡片正常顯示 測試步驟:

1. 用你的 Discord 帳號私訊 Bot：「你好」
2. 觀察回覆
預期結果:
✅ Bot 在 3-5 秒內回應（無超時）
✅ 回覆以 Embed 卡片呈現（不是純文字）
✅ 卡片有紫色側邊條（預設顏色）
✅ 底部顯示「💭 ReMind - 你的 AI 記憶夥伴」
測試變化 - 顏色動態:

再發送：「明天天氣如何？」（包含 ?）
✅ 卡片應該是藍色側邊條

再發送：「太棒了！」（包含 !）
✅ 卡片應該是橘色側邊條
✅ 測試 2: 深度記憶提取（10 分鐘）
目的: 確認事實提取包含完整上下文 測試步驟:

1. 發送：「我的貓叫小白，牠超級可愛」
2. 等待 Bot 回應
3. （可選）等待 5-10 秒，讓記憶系統處理
4. 發送：「我的寵物叫什麼名字？」
預期結果:
✅ Bot 能正確回答「小白」
✅ 可能還會提及「可愛」等你之前說過的細節
檢查 Qdrant（進階）:
登入 Qdrant Cloud Dashboard
查看 n8n_dc_bot collection
確認有新增一筆記錄，內容類似：「用戶的貓叫做小白」
✅ 測試 3: 無效事實過濾（5 分鐘）
目的: 確認 IF 節點正常過濾無意義對話 測試步驟:

1. 發送一些閒聊：
   - 「嗨」
   - 「哈囉」
   - 「天氣真好」
   - 「哈哈哈哈」
預期結果:
✅ Bot 正常回應這些訊息
✅ 但 Qdrant 中不應該新增這些無意義內容
檢查方式:
登入 Qdrant Dashboard
查看最新的記錄時間
確認沒有「嗨」、「哈囉」這類內容被存入
✅ 測試 4: 多用戶記憶隔離（15 分鐘）⭐ 重要
目的: 確認 Metadata Filter 修復成功，A 的記憶不會洩漏給 B 需要準備:
兩個不同的 Discord 帳號（帳號 A 和帳號 B）
測試步驟:

【帳號 A】私訊 Bot：「我叫小明，我喜歡打籃球」
（等待回應）

【帳號 A】：「我喜歡什麼運動？」
預期：Bot 回答「籃球」 ✅

【帳號 B】私訊 Bot：「小明喜歡什麼運動？」
預期：Bot 說「我不知道」或「沒有這個資訊」 ✅

【帳號 B】：「我是誰？」
預期：Bot 說類似「我還不認識你」 ✅
預期結果:
✅ 帳號 A 能讀取自己的記憶
✅ 帳號 B 完全無法讀取帳號 A 的任何資訊
✅ 兩個帳號的記憶完全獨立
✅ 測試 5: 連續對話穩定性（10 分鐘）
目的: 確認系統在多輪對話中穩定運行 測試步驟:

連續發送 10 條訊息，包含：
1. 「我今天很累」
2. 「因為加班到很晚」
3. 「我在做專案報告」
4. 「明天要交給老師」
5. 「你覺得我應該繼續嗎？」
6. 「好吧，我再努力一下」
7. 「對了，我明天要早起」
8. 「7 點要開會」
9. 「你記得我明天幾點開會嗎？」
10. 「謝謝你的陪伴」
預期結果:
✅ 每條訊息都正常回應
✅ 沒有錯誤或超時
✅ 第 9 條訊息時，Bot 能回答「7 點」
✅ 所有回覆都是 Embed 卡片格式

🚨 常見問題排查
問題 1: Bot 沒有回應
檢查:
Discord Trigger 是否啟動
n8n 工作流是否正在運行
問題 2: 回覆仍是純文字，不是 Embed
檢查:
確認有重新載入 JSON 文件
檢查 n8n 中「Send a message」節點的 options 欄位
問題 3: 模型回應超時或 404
解決:
檢查 Groq API 是否有額度
如果 llama-3.3 不支援，改用 llama-3.1-70b-versatile
問題 4: 記憶沒有被保存
檢查:
Qdrant API 憑證是否正確
檢查「過濾無效事實」節點是否正確連接
